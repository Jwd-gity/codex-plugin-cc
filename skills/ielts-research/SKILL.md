# IELTS Research

## 描述

围绕待开发功能，研究 IELTS 方法论和同类工具实现。确保功能设计基于真实的评分标准而非臆测。

## 参数

- feature: string    # 待开发功能描述
- output: string     # 输出文件路径

## 工具

Read, Grep, Glob, Bash, Agent, WebSearch (如可用)

## 步骤

1. **需求拆解**：从功能描述提取关键词，确定涉及的 IELTS 评分维度
2. **项目现状扫描**：
   - `grep` 搜索 IELTS Coach 中相关模块的现有实现
   - 检查 `contracts.py` 中已有的数据模型
   - 检查 `prompts/` 中已有的 LLM 提示词
3. **方法论研究**：
   - IELTS 官方评分标准（TR/CC/LR/GRA 或 FL/PR/LR/GRA）
   - 该功能的"正确实现"应该是什么样
   - 常见陷阱和误区
4. **竞品参考**：
   - 搜索类似 IELTS 工具的实现方式
   - 提取可借鉴的模式
5. **差距分析**：当前项目 vs 目标功能，列出 gap

## 输出格式

```markdown
# 方法论研究：{feature}

## 功能需求拆解
[关键词 + 涉及的 IELTS 维度]

## 项目现状
### 已有能力
- [现有模块/函数/数据模型]
### 缺失能力
- [需要新增的部分]

## IELTS 评分标准参考
[该功能相关的官方评分维度和标准]

## 同类工具参考
[竞品实现方式和可借鉴点]

## 关键发现
[最重要的 3-5 个发现]

## 建议实现方向
[基于研究的初步建议]
```

## 边界

- 研究必须引用来源，不凭空编造评分标准
- 竞品分析只提取模式，不复制代码
- 差距分析必须具体到函数/模块级别
